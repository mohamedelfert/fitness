# E1.9 Billing — Slice A: Plans + Entitlements Foundation — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish the SaaS plans catalog, plan-driven entitlements, and plan-driven AICredit funding — replacing the pre-billing onboarding free-grant stopgap. (Slice A of the E1.9 spec; slices B–D are separate plans.)

**Architecture:** New `Billing` module (Plane A). A `plans` table + seeded `free`/`premium` plans, a `PlanResolver` that returns a Person's plan (Slice A: always the seeded `free` plan; Slice B adds the active-subscription lookup), a `feature` Gate ability reading the plan's `features_json`, and a Billing listener that grants the free plan's `ai_credit_grant` AICredits on `OnboardingCompleted` — retiring AiOrchestration's `GrantFreeAiCredits` stopgap so funding flows through plans.

**Tech Stack:** Laravel 12, PHP 8.2, MariaDB (tests on `fitness_os_test`), PHPUnit, Pint, the auto-wiring `ModuleServiceProvider`, the existing `AiCreditMeter` (AiOrchestration).

## Global Constraints

- PHP 8.2; Laravel 12. Tests run on **MariaDB** (`fitness_os_test`), not SQLite.
- Modules auto-wire: `modules/<X>/Database/Migrations`, `modules/<X>/Routes/api.php` (under `/v1`, `api` group), `modules/<X>/Providers/*ServiceProvider.php`. No manual registration.
- All money is **micro-units** (`price_micros`: 1 USD = 1_000_000). AICredits are integers.
- Every query is **person-scoped**; person-owned resources cross-person → 404.
- **OpenAPI has no CI guard** — after editing `packages/api-contracts/openapi.yaml`, run: `python3 -c "import yaml,re; t=open('packages/api-contracts/openapi.yaml').read(); d=yaml.safe_load(t); s=set(d['components']['schemas']); r=set(re.findall(r'#/components/schemas/(\w+)',t)); print('refs OK' if not r-s else f'DANGLING: {r-s}')"`
- Commits: **no `Co-Authored-By` trailer**. Commit per task. Do **not** push unless the user asks.
- Run from `apps/api/`: tests `vendor/bin/phpunit`, style `vendor/bin/pint --dirty`.

```php
<?php

namespace Modules\Billing\Http;

use App\Http\Controllers\Controller;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;
use Modules\Billing\Models\Plan;

/** Plans catalog (FR-SAS-002) — `GET /v1/plans?audience=b2c`. Active plans for an audience. */
class PlanController extends Controller
{
    public function index(Request $request): JsonResponse
    {
        $audience = $request->query('audience', 'b2c');

        $plans = Plan::where('audience', $audience)->where('active', true)
            ->orderBy('price_micros')->get()
            ->map(fn (Plan $p) => [
                'code' => $p->code,
                'name' => $p->name,
                'price_micros' => $p->price_micros,
                'currency' => $p->currency,
                'interval' => $p->interval,
                'ai_credit_grant' => $p->ai_credit_grant,
                'features' => $p->features_json ?? [],
            ]);

        return response()->json(['data' => $plans]);
    }
}
```

```php
<?php

namespace Modules\Billing\Providers;

use Illuminate\Support\Facades\Gate;
use Illuminate\Support\ServiceProvider;
use Modules\Billing\Services\PlanResolver;
use Modules\Identity\Models\Person;

/**

class BillingServiceProvider extends ServiceProvider

            return ($plan->features_json[$feature] ?? false) === true;
        });
    }
}
```

```php
<?php

namespace Modules\Billing\Listeners;

use Illuminate\Support\Facades\Log;
use Modules\AiOrchestration\Models\AiCreditLedgerEntry;
use Modules\AiOrchestration\Services\AiCreditMeter;
use Modules\Billing\Services\PlanResolver;
use Modules\Identity\Events\OnboardingCompleted;

/**
 * Grants the free plan's AICredit allotment when a Person finishes onboarding (FR-SAS-004),
 * replacing AiOrchestration's pre-billing free_grant stopgap. Plan-driven: the amount is the
 * resolved free plan's ai_credit_grant.
 *
 * Best-effort by design: a failure here must NOT 500 a successful onboarding (worst case is a
 * clean 402 on first AI use). Idempotent via the absence of a prior 'plan_grant' ledger row.
 *
 * ponytail: read-then-grant can race two concurrent first-submits into a double grant. Stakes =
 * one extra free grant; add a unique ledger (wallet_id, reason='plan_grant') index if it matters.
 */
class GrantFreePlanCredits
{
    public function __construct(
        private readonly AiCreditMeter $meter,
        private readonly PlanResolver $resolver,
    ) {}

    public function handle(OnboardingCompleted $event): void
    {
        try {
            $amount = $this->resolver->for($event->person)->ai_credit_grant;
            if ($amount <= 0) {
                return;
            }

            $wallet = $this->meter->walletFor($event->person);
            $alreadyGranted = AiCreditLedgerEntry::where('wallet_id', $wallet->id)
                ->where('reason', 'plan_grant')
                ->exists();

            if (! $alreadyGranted) {
                $this->meter->grant($event->person, $amount, 'plan_grant');
            }
        } catch (\Throwable $e) {
            Log::warning('Free-plan AICredit grant failed; person can top up / will hit 402 on first AI use', [
                'person_id' => $event->person->id,
                'exception' => $e->getMessage(),
            ]);
        }
    }
}
```
